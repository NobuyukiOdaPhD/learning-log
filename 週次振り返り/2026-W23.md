### 個別に学んだこと
#### ドットインストール
- HTMLは文書の意味を示しつけて、コンピュータが正しく活用できるようにするもの
- 最低限の文法
 - <開始タグ>要素</終了タグ>
- タグの種類
 - h1: heading 1 = 見出し1
 - p: paragraph = 段落
 - img: image = イメージを貼り付ける/ 属性: src - ファイルの場所, alt - 代替テキスト
  - <img src = "example.png" alt = "サンプルの代替テキスト">
#### 独習Git
- ブランチの考え方は単に合流させるもの、程度の認識だったが、メイン/トランクから分離して、バージョンを作ることで、開発を継続しつつ、リリースすることができるという概念は今までなかった
- Gitでブランチした場合の「ファイル群がコピーされているわけでもないし、特別なコピーを作るためのリモートサーバーとの相互作用が発生するわけでもない(内部でGitが行うのは、開発の新しい経路あるいは路線を示す特定のコミットと１個のポインタを更新する処理である)」の意味合いが直感的には飲み込めなかった。ローカルレポジトリで作業しているから、新たなコピーは必要ない。
- "hard coding"とは間接的な定義を使わずに数値や文字列を直接コーディングすること
- "sanitized version"とは公開したくない情報を削除したバージョン
- ** ステージングエリアを使うと、自分が提出したい部分だけを選んでコミットできる **
#### Terminal Command
- cd : ホームディレクトリに戻れる
- cd ..: 親ディレクトリに戻れる
- rm -r "ディレクトリorファイル名": ディレクトリまたはファイルを削除
- [↑]/[↓]: コマンドのヒストリに従い、過去のコマンドを呼び出せる
- history：使用したコマンドのヒストリを表示する
- *操作のためのtips*
 - Ctrl + H: Backspace = カーソルの左側の文字を削除
 - Ctrl + D: Delete = カーソルの右側の文字を削除
 - Ctrl + U: 行をすべて削除
 - Ctrl + K: カーソルの右側をすべて削除
 - Ctrl + A: 行の先頭に移動
 - Ctrl + E: 行の末尾に移動
 - コマンドを呼び出す技は"http://www.gnu.org/software/bash/manual/html_node/index.html"
#### Git Command Basics
- 基本構文は"git [スイッチ] <コマンド> <args>"
 - args: arguments=コマンドが認識する引数群
 - e.g.: git -p config --global user.name "name"
- スイッチについては、"-p"=paginate=ページ割りできる
- スイッチに"-"は省略形(-v)、"--"は完全系(--version)の使い分けがある
- git help コマンド名: コマンドに対するヘルプを参照できる
 - pagerで表示されるので、下に送るには[space]、前に戻るには[b]。終了するには[q]。
 - git help glossary: 用語集を確認できる
 - git help -g: Gitガイドの短いリストを表示するもの
##### ファイル操作系
- git ls-files: レポジトリで管理されているファイルのリストを取得
- git blame ファイル名: ファイルの詳細見る。"Q"で終了
##### 確認系
- git --version: gitとのバージョンを確認する
- git config user.name: user nameの確認
- git config user.email: user emailの確認
- git config --list: configurationのすべてのリストの確認
- git log --oneline: レポジトリのコミット履歴を見る
##### 初期設定系
- git config --global user.name "Your Name": Your Nameをuser nameに設定
- git config --global user.email "address@example.com": address@example.comをuser emailに設定
