### 個別に学んだこと - HTML
#### ミッション
- CSSファイルの読み込み
* <head></head>内部にCSSファイルのリンクを入れる -> <link rel="stylesheet" href="style.css">
- 使用したCSSのプロパティ
* "font-family: 'メイリオ', 'Arial', sans-serif;": フォントの指定。Macはメイリオが標準ではないため、'Hiragino Sans'で指定した方が良い
* "max-width: 700px; margin: 0 auto;": 横方向に中央寄せして読みやすい幅にする。-- ただし、画像ファイルは独立して、その幅で表示されているので、700pxをはみ出している状態。-> 画像にmax-width: 100%;を足すと枠の中に収まる。
* "color: #2738F5;": 色指定。ここではHEX表示している。色見本はこちら(https://htmlcolorcodes.com/)
* "line-height: 1.6;": 行間を決める(ここでは1.6倍)
* "background-color: #FEE7FC;": 背景色を決める
- セレクタの使い方
 - idとclassがある。idはページ内で1度だけ、classは何度も使うものに使用=classは複数箇所に当てる
 - class名・id名はケバブケースで記述("lower-hyphen")、idは1ページに1個、styling は class メインで、id は補助的に。<-idは強すぎて、上書きするし、JSで呼び出すときに複数使ってしまうと失敗するから
 - idの例
  - HTML: <p id="sample-id"></p>
  - CSS: #sample-id {}
 - classの例
  - HTML: <p class="sample-class"></p>
  - CSS: .sample-class {}
- 属性の順序は流派がある。
 - "必須属性(href, src, typeなど)">"識別属性(id, nameなど)">"style属性(classなど)">"その他属性"
 - "識別属性(id, nameなど)">"style属性(classなど)">"必須属性(href, src, typeなど)">"その他属性"
- flexboxの使い方
 - flexboxを適用する範囲を<div></div>などで囲み、classを定義
 - cssでclassに対して、{display: flex;}を適用することで、囲んだコンテンツを横長びにする
 - それ以外に使用したプロパティ
  - "align-items: center;": アイテムを中央揃えする(cf. なおjustify-contentは横方向)
  - "gap: 20px;" アイテム間の間隔を入れる
  - "justify-content: flex-start;": flexboxの最初から並べる(それ以外にcenter, space-betweenなど)


#### ドットインストール - CSS, footer, section
- CSS基本構文
 - セレクタ {
  プロパティ: プロパティの値;
 }
 - 例
   img {
    border-radius: 20%;
   }
- CSSを記述する方法はいくつかある
1. <head></head>内部に<style></style>を入れる
- 使用したプロパティ
* "border-width: 1px;": 枠線の幅
* "border-width: gray;": 枠線の色
* "border-style: solid;": 枠線の種類(この場合、実戦)
* "border-radius: 8px;": 枠線の角の丸め
* "padding: 16px;": 枠線の内側に余白を設定する(webサイトでは8の倍数を使うと揃いやすいと言われる)
* "margin-bottom: 8px;" 枠線の外側に余白を設定する(この場合は、下のみ設定)
* "text-align: center;": 中央揃いにする
* "background-image: url(「ファイルのパス」);": 背景画像を設定する
- <footer></footer>: フッター表記を行う
- <section></section>: セクションを区切るための汎用タグ(使用例ではフッターをメインセクションから外すために利用)

### 個別に学んだこと - Git
#### 独習Git
- 差分を確認し、viエディタで直接hunkするファイルを触る
 - "git add -p": 差分を確認する
 - "e": manually edit the current hunk -> viエディタの起動
 - "git reset ファイル名": ステージングエリアに加えた変更を削除する
 - "git checkout -- ファイル名": ファイルを最後にコミットしたバージョンに戻す
 - viエディタの操作
  - "1G": 行の最初に移動
  - "j": 下の行に移動
  - "dd": 行を削除
  - ":wq": viエディタを終了(忘れて焦りそうなので、再度書いておく！)
- 忘れていたコマンド
 - git init
 - git ls-files
 - git add ファイル名
